=== SAMPLE DATA HIERARCHY ===

1 User (STUDENT) → 1 Student → Multiple Internships, Projects, Co-curriculars
                              → 1 Career Details
                              → 1 Personal Problem
                              → Multiple Semesters → Multiple Subjects
                              → Multiple Messages (sent/received)
                              → Multiple Requests
                              → Multiple Meetings

1 User (FACULTY) → 1 Faculty → Multiple Mentoring Students
                              → Multiple Meetings
                              → Optional HOD relation

1 User (HOD) → 1 HOD → Multiple Requests to approve
                     → Multiple Meetings

1 User (ADMIN) → 1 Admin

=== NOTES ===
- All passwords should be hashed using bcrypt before storing
- All IDs are auto-generated CUIDs (Collision-resistant Unique IDs)
- Many-to-many relationships exist between:
  * Students and Internships
  * Students and Projects
  * Students and Co-curriculars
  * Students and Semesters
  * Students and Faculties (mentoring relationship)
  * Semesters and Subjects
- One-to-one relationships:
  * User to Student/Faculty/HOD/Admin
  * Student to Career Details
  * Student to Personal Problem
- One-to-many relationships:
  * Student to Requests
  * HOD to Requests (approval)
  * User to Messages (sender/receiver)
